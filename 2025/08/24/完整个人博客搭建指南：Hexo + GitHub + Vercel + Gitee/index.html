完整个人博客搭建指南：Hexo + GitHub + Vercel + Gitee

发表于2025-08-24|更新于2025-08-24
|浏览量:

下面是一个从零开始的完整教程，教你如何使用 Hexo 创建博客，并部署到 GitHub Pages 和 Vercel，同时将代码备份到 Gitee。

第一阶段：Hexo + GitHub Pages 搭建

步骤 1: 环境准备

  1. 安装 Node.js

    • 访问 Node.js 官网
    • 下载并安装 LTS（长期支持版）
    • 安装完成后，打开终端/命令提示符，验证安装：

  2. 安装 Git

    • 访问 Git 官网
    • 下载并安装 Git
    • 安装后配置全局用户信息：

步骤 2: 安装并初始化 Hexo

  1. 安装 Hexo CLI

  2. 初始化博客项目

  3. 本地预览

    在浏览器中访问 http://localhost:4000 查看博客

步骤 3: 创建 GitHub 仓库

  1. 登录 GitHub

    • 访问 GitHub 并登录

  2. 创建新仓库

    • 点击右上角 “+” → “New repository”
    • 仓库名格式: 你的用户名.github.io (例如: johnsmith.github.io)
    • 选择 “Public”
    • 勾选 “Initialize this repository with a README”
    • 点击 “Create repository”

    • 打开浏览器访问 https://你的用户名.github.io
    • 你的博客已经上线了！

步骤 5: 日常写作流程

  1. 创建新文章

  2. 编辑文章

    • 使用任何文本编辑器编辑 source/_posts/文章标题.md

  3. 本地预览

  4. 部署更新

第二阶段：添加 Vercel 部署

步骤 1: 注册 Vercel

  1. 访问 Vercel 官网
  2. 使用 GitHub 账号注册/登录

步骤 2: 部署到 Vercel

  1. 在 Vercel 中导入项目

    • 点击 “Add New…” → “Project”
    • 从 GitHub 仓库列表中选择你的博客仓库
    • 点击 “Import”

  2. 配置项目设置

    • Framework Preset: 选择 “Hexo”
    • Build Command: 输入 npm run build 或 hexo generate
    • Output Directory: 输入 public
    • 其他设置保持默认

  3. 部署

    • 点击 “Deploy”
    • 等待部署完成

  4. 访问你的 Vercel 博客

    • 部署完成后，Vercel 会提供一个类似 xxx.vercel.app 的域名
    • 点击即可访问你的博客

步骤 3: 配置自定义域名（可选）

  1. 在 Vercel 项目仪表板中
    • 点击 “Settings” → “Domains”
    • 输入你的自定义域名
    • 按照提示配置 DNS 记录

第三阶段：添加 Gitee 备份

步骤 1: 创建 Gitee 账户和仓库

  1. 访问 Gitee 并注册/登录
  2. 创建新仓库
    • 点击 “+” → “新建仓库”
    • 仓库名: 与 GitHub 仓库同名 (例如: 你的用户名.gitee.io)
    • 选择 “公开”
    • 点击 “创建”

步骤 3: 推送代码到两个仓库

  1. 首次推送 如果push到github时有下方报错 解决方案： 使用clash verge代理
  2. 使用 HTTP 代理,端口需要去clash verge中看具体使用的是哪个端口
  3. 生成新的 Classic Token（ghp_ 开头，repo 权限）
    生成 Personal Access Token
    登录 GitHub 网站：https://github.com
    点击右上角头像 → Settings
    左侧菜单 → Developer settings
    再左侧 → Personal access tokens → Tokens (classic) 或 Fine-grained tokens
    推荐先用 Classic Token（兼容性更好）
    点击 Generate new token → Generate new token (classic)
    填写信息：
    Note: 比如 git-push-token
    Expiration: 选择过期时间（建议 30 天或 90 天）
    Scopes: 勾选 repo（这是最关键的权限）
    点击 Generate token
    ⚠️ 复制生成的 token（一串字符，如 ghp_XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX）
    🔐 仅显示一次，关闭后就看不到了

方案 B: 使用 GitHub Actions 自动同步到 Gitee

  1. 创建 GitHub Token

    • 在 GitHub → Settings → Developer settings → Personal access tokens
    • 生成新 token，勾选 repo 权限

  2. 在 Gitee 创建私人令牌

    • 在 Gitee → 设置 → 私人令牌
    • 生成新令牌，勾选 projects 权限

  3. 在 GitHub 仓库中添加 Secrets

    • 进入你的 GitHub 仓库 → Settings → Secrets → Actions
    • 添加以下 Secrets:
      • GITEE_TOKEN: 你的 Gitee 私人令牌
      • GITEE_USERNAME: 你的 Gitee 用户名

总结

你现在已经拥有了一个完整的博客系统：

  1. 写作：使用 Hexo 在本地用 Markdown 写作
  2. 部署：
    • GitHub Pages: 主要托管平台
    • Vercel: 全球 CDN 加速，自动化部署
  3. 备份：代码同步到 Gitee，确保数据安全
  4. 自动化：通过 GitHub Actions 实现自动同步

访问方式：

  • GitHub Pages: https://你的用户名.github.io
  • Vercel: https://你的项目名.vercel.app (或你的自定义域名)
  • Gitee: 仅作为代码备份，不直接用于访问

这个方案结合了多个平台的优点，提供了高速访问、自动化部署和数据备份的完整解决方案。如果在实施过程中遇到任何问题，可以随时提问！
